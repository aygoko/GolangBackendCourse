Задание: Разработка HTTP-сервиса для управления ресурсоемкими задачами

---

### **Цель задания**  

Реализовать HTTP-сервис, который принимает задачи от пользователей, управляет их состоянием и предоставляет результаты. Сервис должен быть масштабируемым, гибким и соответствовать требованиям эффективного расходования вычислительных ресурсов.

---

### **Требования к функциональности**  

1. **API-интерфейс:**  
   - **POST /task**  
     - Загрузка задачи на обработку.  
     - Генерация уникального `task_id` (UUIDv4).  
     - Возврат `task_id` пользователю.  
     - Имитация обработки задачи (используйте `time.Sleep()` для задержки).  
     - Результат задачи — случайные данные (например, строка "Результат: [мусор]").  

   - **GET /status/{task_id}**  
     - Возврат статуса задачи в формате JSON:  
       ```json  
       { "status": "in_progress" | "ready" }  
       ```  

   - **GET /result/{task_id}**  
     - Возврат результата задачи (если задача завершена).  

2. **Хранение данных:**  
   - Используйте RAM-хеш-таблицу (например, `map[string]interface{}`) для хранения задач.  
   - Реализуйте интерфейс `Storage` (например, `TaskStorage`), который абстрагирует доступ к данным.  
   - Структура данных должна быть легко заменяема на БД в будущих заданиях.  

3. **Безопасность:**  
   - На данном этапе не требуется аутентификация, но код должен быть готов к её интеграции в будущем.  

---

### **Технические требования**  
- **Язык:** Go (версия 1.20+).  
- **Фреймворк:** Используйте `Fiber` для реализации HTTP-сервера.  
- **Документация:**  
  - Составьте Swagger-документацию для API с помощью пакета `swaggo`.  
  - Добавьте комментарии в код для генерации документации (пример: `// @Summary Получить статус задачи`).  
- **Контейнеризация:**  
  - Напишите `Dockerfile` для сборки и запуска сервиса.  
  - Убедитесь, что контейнер запускается без ошибок (например, через `docker run`).
